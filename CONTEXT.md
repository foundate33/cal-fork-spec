# Cal Fork

Приложение для планирования встреч: хосты создают типы событий, устанавливают регулярную доступность, а букеры бронируют слоты через публичную ссылку.

## Language

**Host**:
Человек, который создаёт типы событий, управляет своей доступностью и принимает встречи.
_Avoid_: Author, User

**EventType**:
Шаблон встречи (название, длительность, zoom-ссылка, slug), создаваемый хостом. EventType занимает календарь хоста. Каждый EventType ссылается ровно на один AvailabilityRule.
_Avoid_: Meeting type, event template

**AvailabilityRule**:
Повторяющееся еженедельное правило (дни недели, время начала/окончания), определяющее когда хост доступен для встреч. Все времена хранятся в UTC.
_Avoid_: Availability, schedule, timezone

**Booker**:
Человек, который бронирует встречу с хостом через публичную страницу бронирования.
_Avoid_: Guest, invitee, participant

**Slot**:
Вычислимое временное окно для бронирования. Слоты не хранятся — они порождаются пересечением AvailabilityRule и длительности EventType за вычетом уже забронированного времени.
_Avoid_: Time slot, window

**Booking**:
Факт бронирования: букер выбирает Slot и создаёт запись, которая занимает календарь хоста. Букинг пересекается по времени с другими букингами того же хоста независимо от EventType — пересечения недопустимы.
_Avoid_: Reservation, appointment
